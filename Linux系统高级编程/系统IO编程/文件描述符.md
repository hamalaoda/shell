### *文件描述符*

起因：在Linux系统中，一切皆是文件。

所以：有文件描述符（非负整数）来标识和管理<font color='red'>一个打开的资源</font>。 通过文件描述符来操作

- 在进程启动成功，为当前进程自动分配三个特殊文件描述符

  \#define STDIN_FILENO 0 //   标准<font color='blue'>输入</font>的文件描述符

  \#define STDOUT_FILENO 1 //标准<font color='blue'>输出</font>的文件描述符

  \#define STDERR_FILENO 2 // 标准<font color='blue'>错误</font>的文件描述符



<font color='red'> fd 是 test.txt 在当前进程中的 “唯一标识”，内核通过 fd 找到 test.txt 的元数据（如存储位置、读写权限等），因此所有对 test.txt 的操作（读写、指针移动等）都需通过 fd 传递给内核，由内核完成实际的磁盘交互</font>
